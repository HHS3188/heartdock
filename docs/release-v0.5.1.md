# HeartDock v0.5.1 发布说明

## 概述

v0.5.1 是 v0.5.0 之后的稳定性修复版本，重点改进鼠标交互状态切换、纯享模式 hit-test 和全局快捷键响应。

## 主要修复

### 修复/缓解鼠标交互状态切换问题（#48）

在部分 Windows 环境下，通过 `Ctrl + Shift + H` 切换鼠标交互状态时可能响应慢、状态不同步，或在纯享模式中触发异常。v0.5.1 进行了以下稳定性修复：

- **主进程** 将全局禁止交互和纯享模式局部 hit-test 透传拆开，避免状态互相覆盖
- **主进程** 增加更可靠的 `Ctrl + Shift + H` 长按保护，避免自动重复触发导致状态快速反转
- **渲染进程** 纯享模式 hit-test 只在全局禁止交互关闭时工作
- **渲染进程** 修复纯享模式 cleanup 使用旧状态覆盖主进程状态的问题

修复后用户测试确认当前体验正常。

### 新增鼠标禁止交互模式（#54）

`Ctrl + Shift + H` 现在开启的是鼠标禁止交互模式：

- 开启后，HeartDock 会拦截鼠标点击
- 点击不会操作 HeartDock，也不会点到底层窗口
- 禁止交互状态下点击 HeartDock 会提示“当前处于禁止交互状态，按 Ctrl + Shift + H 解锁 HeartDock”
- 再次按 `Ctrl + Shift + H` 可解锁并恢复正常交互

## 下载

## 说明

- 本版本改变了旧的鼠标穿透语义：全局快捷键现在用于禁止 HeartDock 交互，并拦截点击，不再把点击直接传到底层窗口。
- 纯享模式仍会在全局禁止交互关闭时使用局部 hit-test，让心率本体可交互、透明区域尽量不挡鼠标。
- BLE 心率设备仍为实验性功能。
- 仅支持 Windows。
