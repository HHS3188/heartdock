# 已知问题说明

本文记录 HeartDock 当前预发布版本中已经确认或需要继续观察的问题。

当前版本：**v0.9.0**

## 鼠标交互状态切换稳定性（v0.5.1 已修复/缓解）

### 现象

在部分 Windows 环境下，频繁通过 `Ctrl + Shift + H` 切换鼠标交互状态时，HeartDock 可能出现响应慢、状态不同步、纯享模式下仍可交互，或关闭时偶发退出。

### v0.5.1 修复

v0.5.1 对该问题进行了稳定性修复：

- 主进程将全局“禁止交互”和纯享模式局部 hit-test 透传拆开，避免互相覆盖状态。
- `Ctrl + Shift + H` 增加更可靠的长按保护，避免自动重复触发导致状态快速反转。
- 全局禁止交互开启后，HeartDock 会拦截鼠标点击，不会操作 HeartDock，也不会点到底层窗口。
- 禁止交互状态下点击 HeartDock 会提示“当前处于禁止交互状态，按 Ctrl + Shift + H 解锁 HeartDock”。
- 纯享模式下仅在全局禁止交互关闭时，才允许心率本体可交互、透明区域局部透传。

修复后用户测试确认当前体验正常。v0.9.0 又进一步拆分透明空白区域 hit-test 与可见内容拦截行为。如仍遇到此问题，请提交 GitHub Issue 并注明 Windows 版本、显示器 DPI、是否多显示器和复现步骤。

### 影响范围

- 主要影响鼠标交互状态、纯享模式和全局快捷键切换场景。
- 模拟心率、手动心率、启动页、显示样式、日间 / 黑夜模式、普通设置页不受直接影响。

## 后续待观察

- 纯享模式增强置顶会尽量覆盖全屏 / 无边框全屏应用，但受 Windows、游戏反作弊、独占全屏和受保护窗口策略影响，仍可能存在特定进程无法覆盖的情况。
- 不同 Windows 版本、多显示器、高 DPI 和特殊显卡驱动环境下，透明窗口和局部 hit-test 行为可能存在差异。
- v0.9.0 的自动更新能力从安装 v0.9.0 后对后续版本生效，v0.7.0 及更早版本无法自动升级到 v0.9.0。
- 自动更新依赖 GitHub Release 元数据；网络不可用时会提示稍后手动检查，不应长期阻塞进入应用。
- BLE 心率设备仍为实验性功能，设备兼容性取决于是否支持标准 BLE Heart Rate Service。
- 纯享心率记录只保存本次纯享会话，不保存历史记录；BLE 未连接或心率显示为 `--` 时不会写入无效样本。
