# 开发路线图

本文记录 HeartDock 的阶段目标、当前进度和后续计划。

HeartDock 当前处于早期预发布阶段。当前重点是先把 Windows 桌面心率悬浮窗、标准 BLE 心率读取、纯享显示模式、显示样式自定义、显示框样式预设和基础开发流程打磨稳定，再逐步考虑图片背景、安装包、目标窗口跟随和更完整的用户体验。

## v0.5.1 — 稳定性修复

v0.5.1 是在 v0.5.0 基础上的稳定性修复版本。

主要修复：

- [x] 修复/缓解鼠标交互状态切换异常和偶发退出（#48）
  - 主进程拆分全局禁止交互和纯享模式局部 hit-test 透传

- [x] `#54 增加交互锁定模式`：并入 v0.5.1 为鼠标禁止交互模式，开启后拦截点击且不点到底层窗口

## 暂不优先处理的方向

以下内容暂时不作为近期重点：

- 小米手环私有协议适配
- 厂商私有认证密钥流程
- 跨平台 macOS / Linux 支持
- 自动更新
- 云端同步
- 用户账号系统

这些方向不是完全放弃，而是当前阶段优先级低于标准 BLE、Windows 悬浮窗体验、自定义显示体验和安装包发布。
