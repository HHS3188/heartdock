## 修改安全规则

- 每次改代码前确认 git status。
- 每次改代码前记录基线提交 SHA。
- 大改动前说明影响范围。
- 影响已有正常功能时，优先回退到修改前正常状态。
- 未经用户确认，不执行破坏性 git 操作。
