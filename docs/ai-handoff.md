- 纯享模式增强置顶已通过用户体验确认，但特定独占全屏、游戏反作弊或受保护窗口仍可能限制覆盖。

- BLE 心率设备仍为实验性功能。

- 纯享心率记录只保存本次会话报告，不保存历史记录。

## 修改安全规则

- 每次改代码前确认 git status。
- 每次改代码前记录基线提交 SHA。
- 大改动前说明影响范围。
- 影响已有正常功能时，优先回退到修改前正常状态。
- 未经用户确认，不执行破坏性 git 操作。

## 交接总结规则

- 默认简短：优先 5-8 行说清做了什么、改了什么、检查结果、Git 状态、下一步。
- 大改动、出错、高风险、准备 commit/push/release 时，必须输出详细阶段总结。
- Claude Code 自行判断详细程度，不强制固定模板。
