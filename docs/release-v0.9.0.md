# HeartDock v0.9.0 发布说明

## 概述

v0.9.0 是正式候选体验增强版本，重点补齐首次使用引导、自动更新、诊断导出、独立心率报告窗口，以及透明窗口鼠标命中行为。

## 主要更新

### 启动与新手引导（#63）

- 新增启动红心动画。
- 启动页会先进行更新检查，进入按钮至少等待 3 秒后才可使用。
- 新增分步新手引导，覆盖数据源、显示区、纯享模式、动态显示、记录导出和系统工具。
- 新手引导使用单一“下一步 / 完成”控件；完成后不再自动弹出。
- 设置页提供重新打开新手引导的入口。

### 设置帮助与布局（#63）

- 主要设置项新增圆形帮助提示。
- 帮助提示使用简短中文说明用途和影响。
- hover/focus 可自动展示，触控环境可点击查看。
- 设置页工具区整合手动检查更新、导出诊断信息、配置导入导出和反馈入口。

### 自动更新（#63）

- 新增 `electron-updater` 依赖。
- electron-builder 增加 GitHub publish 元数据。
- 启动页和设置页均可触发更新检查。
- 更新弹窗展示版本号、简短摘要、下载进度和发布页链接。
- `<1.0.0` 内的更新允许暂不更新；跨到 `>=1.0.0` 大版本时按强制更新处理。

> 自动更新能力从安装 v0.9.0 后对后续版本生效。v0.7.0 及更早版本仍需要手动下载安装包升级到 v0.9.0。

### 鼠标与透明窗口 hit-test（#63）

- 重新整理鼠标禁止交互与局部 hit-test 的关系。
- 鼠标禁止交互开启后，可见 HeartDock 内容区域会拦截点击，不操作 HeartDock，也不点到底层窗口。
- 透明空白区域会尽量透传到底层窗口，减少“虚空挡板”。
- 保留 `Ctrl + Shift + H` 的用户语义，用于开启 / 关闭鼠标禁止交互。
- 主窗口和报告窗口会限制在屏幕边缘内，左右至少保留约 10% 宽度可见，上下至少保留约 5% 高度可见，避免窗口被拖到找不到的位置。

### 纯享心率报告（#63）

- 纯享心率记录报告改为独立窗口。
- 报告窗口拥有独立最小尺寸和滚动策略，不再被主设置窗口尺寸压缩。
- 报告窗口支持关闭和 PNG 导出。

### 诊断导出（#63）

- 设置页新增“导出诊断信息”。
- 导出文件为带日期时间的 `.json`。
- 内容包含版本、Windows / Electron / Node 信息、心率来源、纯享 / 置顶 / 鼠标禁止交互状态、显示样式、动态效果、更新状态和最近关键运行状态。
- 导出内容做脱敏处理，不包含本地路径、设备名、原始配置全文或隐私字段。

- `npm run build` 通过。
- `npm run dist` 通过，已生成 v0.9.0 NSIS 安装包，并确认 `latest.yml`、blockmap 和 exe 资产齐全。

## 说明

- BLE 心率设备仍为实验性功能。
- 仅支持 Windows。
- 当前安装包未购买代码签名证书，Windows 可能提示“未知发布者”或 SmartScreen 风险提示。请只从本项目 GitHub Release 页面下载。
- 透明窗口 hit-test 在多显示器、高 DPI、受保护窗口和部分全屏程序下仍可能存在系统差异。
